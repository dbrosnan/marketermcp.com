# Governance

## Project lead

This project is currently maintained by **Drew Brosnan** ([@dbrosnan](https://github.com/dbrosnan)).

## Decision making

- Day-to-day decisions are made by the project lead.
- Significant architectural decisions are documented as Architecture Decision Records (ADRs) in `docs/adr/`.
- Community input is welcomed via issues and pull requests.

## Roles

| Role | Responsibilities |
|---|---|
| **Maintainer** | Merge PRs, manage releases, triage issues, enforce standards |
| **Contributor** | Submit PRs, report bugs, propose features |

## Becoming a maintainer

Contributors who demonstrate sustained, high-quality contributions may be
invited to become maintainers at the project lead's discretion.

## Changes to governance

This document may be updated via pull request. Significant changes require
discussion in an issue before merging.
