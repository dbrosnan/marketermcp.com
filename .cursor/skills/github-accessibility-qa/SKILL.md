# GitHub-Inspired Accessibility QA Skill

## Use when

- Building or updating public website pages.
- Running release QA before launch.

## Outcome

Finds and fixes common accessibility issues early in design and implementation.

## Steps

1. Review semantic HTML structure:
   - heading order
   - form labels
   - link clarity
2. Validate color contrast and focus visibility.
3. Check keyboard interaction paths for all CTA and nav items.
4. Add or update an accessibility checklist in PR notes.
5. Recommend CI automation paths with:
   - `dequelabs/axe-core`
   - `pa11y/pa11y-ci`
