# Support

## Getting help

- **Documentation**: Browse the [`docs/`](docs/) directory for product docs, quickstart guides, and brand standards.
- **Issues**: Search [existing issues](https://github.com/dbrosnan/marketermcp.com/issues) before filing a new one.
- **Discussions**: For questions, ideas, and general conversation, use [GitHub Discussions](https://github.com/dbrosnan/marketermcp.com/discussions) (when enabled).

## Filing a bug

Use the **Bug report** issue template. Include:

1. What you expected to happen.
2. What actually happened.
3. Steps to reproduce.
4. Environment details (OS, Node version, browser if applicable).

## Feature requests

Use the **Feature request** issue template. Describe the problem you're trying to solve before proposing a solution.

## Security issues

See [SECURITY.md](SECURITY.md) for responsible disclosure instructions. Do **not** open a public issue for security vulnerabilities.

## Contact

For anything not covered above, reach out at **drew@marketermcp.com**.
